Module 02
Exam Form — Backend
Node.js / Express — Candidate, Institute & RBTE exam form application workflows
CCommercial / Functional Changes
  • Elective completeness rule: Regular form now persists selected electives with flag='Y' and remaining/unselected electives as optional with flag='O' in exam_subject_s via insertOptionalElectiveSubjects.
  • Idempotent elective backfill: NOT IN guard on existing exam_subject_s elective records prevents duplicates on rerun/reprocessing.
  • Exemption handling: Main subjects updated to flag='E' for exempted papers via updateExemptionFlag.
  • Feature-window enforcement: Apply/confirm/cancel gated by centralized config flags (e.g. START_EXAMFORM_W_*) — hard close/open control for operations.
  • Role-based workflows: Separate flows for Candidate / Institute / RBTE with distinct routes and access control.
BBusiness / Operational Impact
  • Eliminated cases where electives were absent in exam_subject_s after selection — now guaranteed: selected Y + remaining O.
  • Reprocessing is duplicate-proof (idempotent insert), reducing manual DB cleanup during retries to zero.
  • Config-based gating reduces "link open/close" manual interventions to zero-code operational toggles.
  • Set-based INSERT … SELECT for regular flow subjects reduces per-subject DB round trips significantly.
  • In-module caching (Map) for repeated lookups: paper-code block and exam-center resolution reduce repeated DB hits.
TTechnical Enhancements / Upgrades
Node.js Express JWT Auth Joi Validation MySQL
  • Subject inserts standardized as INSERT … SELECT from course_sub_marks / course_sub_marks_lateral.
  • paper_code mapped consistently from paper_code_eq in marks master during inserts.
  • Copy-table and BK-table write paths added (exam_subject_s_copy, exam_subject_w_bk) controlled via EXAM_FORM_W_BK config.
  • New endpoints: /beforeApply, /apply, /applyNext with X-form variants and shared controller dispatch.
  • applyNext implements copy-then-delete on re-apply: move rows to *_copy, delete from live table, then reinsert.
AArchitecture Improvements
  • Exam form split by domain roles: resources/candidate|institutes|rbte/examform/*
  • Shared core under resources/shared/*.
  • Query strings and business logic separated (controller → utils/service → query map).
  • Centralized SQL in shared/query.js and utils/functions/examform/query.js.
  • Dynamic subject table resolution by form type + block.
SSecurity Enhancements
  • JWT-based role checks: candidateAuth, instituteAuth, rbteAuth per route.
  • Strict Joi schemas: enrollment format, course/year/master codes, appear code, exam_no length, block validation.
  • Feature flag gating (requireConfig): hard 403 + explicit disabled flag list prevents unauthorized access outside configured window.

Module 05
E-Marksheet — Backend
Node.js / Express — Examiner confirmation, PDF generation, ImmuDB audit, bulk UPSERT workflows
CCommercial / Functional Changes
  • S26 season rollout: Updated confirmation artifacts to follow S26 naming convention — PDF naming _s26.pdf + supporting table prefix updates.
  • Confirmation workflow hardening: Examiner confirmation now performs lock-check + already-confirmed check concurrently via Promise.all, preventing double-confirm or inconsistent state during peak operations.
  • PDF generation resilience: Confirmation flow treats PDF-service timeouts as recoverable by verifying PDF existence on disk before failing — reduces operational disruptions during load spikes.
  • ImmuDB lock write correctness: Fixed region-partition lock table name construction (removed underscore before region code) to ensure writes land in the correct final lock table.
  • Data integrity enforcement: Bulk UPSERT now rejects invalid rows where seat_no/sr_no are missing — prevents broken marks_uuid and downstream reconciliation issues.
  • Admin access enablement: Added http://emsadmin.4pindia.com to allowed origins to support admin-side operational workflows.
BBusiness / Operational Impact
  • PDF timeouts no longer automatically fail confirmation if the PDF is already on disk — reduces avoidable rework and escalations.
  • Parallel prechecks reduce confirmation pre-validation latency from sequential (t1+t2) to approximately max(t1,t2) for that step.
  • Prevents bad lock writes (wrong table) and invalid marks_uuid rows, reducing manual cleanup and retry cycles.
  • Cleaner production observability: structured tracing added during debugging, noisy logs removed after.
TTechnical Enhancements / Upgrades
  • PDF service call timeout increased: 15s → 45s, plus timeout-recovery logic via disk existence check.
  • Parallelized independent checks using Promise.all.
  • Added consistent error logging utility for lock/confirmation failures.
  • Base64-encoded binary/control bytes for enrcrypt_marks before ImmuDB SQL literal transport.
  • Bulk write refactor: excluded DB-derived marks_uuid from bulk column lists; generate deterministically from seat_no-sr_no[-C].
  • Corrected target lock table naming: ${ORPR_OUTPUT_FINAL_LOCK}${regionCode} (region partition).
